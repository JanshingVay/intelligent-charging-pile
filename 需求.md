# 二、 核心业务规则与算法逻辑 (必须严格实现)

## 1. 计费规则
* **单位电价（按时间段不同）**:
    * 峰时: 1.0元/度 (10:00~15:00, 18:00~21:00)
    * 平时: 0.7元/度 (7:00~10:00, 15:00~18:00, 21:00~23:00)
    * 谷时: 0.4元/度 (23:00~次日7:00)
* **服务费单价**: 固定 0.8元/度。
* **总费用计算公式**: 总费用 = (单位电价 * 实际充电度数) + (服务费单价 * 实际充电度数)。注意跨时间段充电时的分段计费逻辑。

## 2. 基础调度策略 (正常排队)
* 选取等候区排队号码和该充电桩模式匹配的第一辆车。
* 加入队列的依据：在同模式充电桩中，选择让当前车辆**完成充电所需总时长（等待时间 + 自身充电时间）最短**的桩。
    * 等待时间 = 选定充电桩队列中所有车辆完成充电时间之和。
    * 自身充电时间 = 请求充电量 / 充电桩功率。

## 3. 异常处理：修改与取消请求
* **等候区**:
    * 修改模式：重新生成排队号，排至新模式队列队尾。
    * 修改电量：更新请求，原排队号不变。
* **充电区**:
    * 不允许修改任何请求参数。
* **取消**: 等候区和充电区均允许取消。充电区取消需立刻结算已充费用并释放充电桩。

## 4. 故障处理与重调度逻辑 (SchedulingController)
发生故障时，当前车辆停止计费并生成部分详单，暂停等候区叫号。实现以下三种调度策略函数：
1.  **优先级调度 (`handlePriorityDispatch`)**: 收集故障桩队列中所有等待车辆，遍历它们，利用 `findIdlePile` 寻找同类型其他正常桩的空位并分配。待全部调度完毕后，恢复等候区叫号。
2.  **时间顺序调度 (`handleTimeSeqDispatch`)**: 将故障队列中的车辆与**其他同类型充电桩中尚未充电的车辆**合为一组，按照原始 `QueueNumber` 上的生成时间（排队号码先后）重新排序。清空原有排队，按照新顺序依次分配到正常的同类充电桩队列中，最后恢复叫号。
3.  **充电中故障恢复 (`handleRecoveryDispatch`)**: 当故障桩被管理员修复重新 `Available`，暂停等候区叫号。获取其他同类型桩中尚未充电的车辆，按排队号先后顺序重新分配（此时修复好的桩可分担压力），最后恢复叫号。

## 5. 扩展调度策略 (选做项，必须实现)
1.  **单次调度总时长最短 (`calcMultiSpotOptimal`)**: 当充电区出现多个空位，一次叫多个号。约束：按模式分配；需满足进入充电区的多辆车完成充电**总时长之和最小**（可使用贪心算法或穷举分配组合）。
2.  **批量调度总时长最短 (`calcBatchOptimal`)**: 当到达站点的车辆等于全部车位数量 (N + 5*M) 时触发。忽略快慢充模式限制（所有车可分配任意桩），忽略先后顺序。目标函数：使这批车辆完成充电的 **总时长之和最小** `Min(∑(各车等待时间 + 自身充电时间))`。
